# پیشنهاد ریفکتور: پلی‌لیست‌های کانال (`ChannelPlaylistPanel` و API)

این سند ایده‌های مشخص برای مرحلهٔ بعد توسعه است؛ پیاده‌سازی در همین PR انجام نشده است.

## ۱. معماری داده و state

- **تفکیک هوک‌ها**: `useChannelPlaylists`, `useChannelQueuePreview`, `usePlaylistDragDrop` تا کامپوننت نازک شود و تست واحد روی منطق ممکن شود.
- **منبع حقیقت**: الان ترکیب REST + WebSocket است؛ قرارداد مشخص کنید: «صف کانال از کجا خوانده می‌شود» (فقط `/queue` یا فقط snapshot سوکت) تا race و فلیکر کم شود.
- **کش SWR/React Query** برای `GET /playlists?channel=` و آیتم‌ها با `staleTime` مناسب و invalidation بعد از mutating.

## ۲. UX پلی‌لیست کانال

- **نوار ابزار ثابت**: جستجو، فیلتر (مال من / کانال)، مرتب‌سازی، «shuffle به صف» در یک سطر sticky.
- **کارت پلی‌لیست**: پیش‌نمایش تعداد ترک، آخرین به‌روزرسانی، اکشن‌های شفاف (Play در کانال، ویرایش، حذف با تأیید).
- **خالی بودن**: empty state با CTA (آپلود، shuffle، لینک به تب queue).
- **دسترسی**: نشان دادن صریح «فقط خواندنی» برای عضو عادی vs مدیریت برای مود/اونر.

## ۳. کشیدن و رها کردن (Reorder)

- استفاده از **dnd-kit** یا `@hello-pangea/dnd` به‌جای HTML5 DnD خام برای موبایل و دسترسی بهتر.
- **Optimistic UI** + rollback روی خطای API؛ نوار باریک وضعیت ذخیرهٔ در حال انجام.

## ۴. عملکرد

- **مجموعه‌ای از آیتم‌ها**: virtualize لیست بلند (`@tanstack/react-virtual`) اگر تعداد آیتم‌ها زیاد است.
- **بارگذاری تنبل** `track_detail` فقط وقتی ردیف visible است (در صورت پشتیبانی API).

## ۵. API و بک‌اند (در صورت نیاز)

- **فیلتر سمت سرور**: `?search=&owner=` روی playlist items برای کاهش payload.
- **تراکنش**: reorder دسته‌ای در یک endpoint (`PATCH bulk`) برای کم کردن round-trip.
- **سقف اندازهٔ پلی‌لیست** روی کانال (مثل shuffle) برای جلوگیری از سوءاستفاده.

## ۶. تست و کیفیت

- **قرارداد**: تست e2e سبک (Playwright) برای «ایجاد پلی‌لیست → افزودن ترک → play در کانال».
- **Storybook** (اختیاری) برای حالت‌های empty / loading / error کارت‌ها.

## ۷. اولویت‌بندی پیشنهادی

1. تفکیک هوک + قرارداد منبع حقیقت صف  
2. تب/نوار ابزار و empty state  
3. dnd-kit + optimistic reorder  
4. مجازی‌سازی در صورت نیاز واقعی پروفایل  

با این ترتیب هم risk کم است هم ارزش UX زودتر به دست می‌آید.
